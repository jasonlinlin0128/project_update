# 資料庫查詢 Skill

## 步驟 1：資料準確性警告（必做，不可跳過）

在做任何事之前，先用 AskUserQuestion 顯示以下警告並詢問：

> ⚠️ **資料準確性提醒**
>
> Firebase 中的資料不一定是最新的。常見原因：
> - 專案進度欄位沒有即時更新
> - 部分資料是手動匯入，可能有落差
>
> **在這個前提下，你想查什麼？**
> 請描述你的問題，例如：
> - 「哪些專案進度低於 50% 但截止日在本月內？」
> - 「目前有哪些還在進行的接洽案？」
> - 「列出所有負責人是王小明的專案」

---

## 步驟 2：確認查詢意圖

根據使用者的描述，**在執行任何查詢前**再用 AskUserQuestion 確認：

> 「我理解你想查：[重述使用者的問題]
>
> 這樣正確嗎？如果資料有落差，結果僅供參考。」

---

## 步驟 3：執行查詢

### 方式 A：先拉完整資料，再用 jq 過濾

```bash
node scripts/firebase-utils.mjs list-projects
```

再根據使用者的條件用程式邏輯過濾，不需要真的寫 Firestore query。

## 支援的查詢類型範例

| 使用者說的 | 對應 collection |
|-----------|----------------|
| 哪些專案... | `projects` |
| 有哪些例行公事... | `routines` |
| 論壇 / 分享會... | `forums` |
| 活動 / 出差... | `events` |
| 接洽中的廠商... | `coordination` |
| 最近的會議... | `meetings` |

---

## 注意事項

- 永遠先問，不要直接查
- 若結果異常（如所有進度都是 0），主動提醒使用者這可能是資料未同步的問題
- 不要嘗試寫入任何資料，這個 skill 只做讀取
